# WhatsApp Automation Integration & Setup Guide

This guide explains how to configure and deploy the self-hosted **Baileys WhatsApp API Gateway** (Node.js/Express) and connect it to your InsureDesk CRM on production (`bimaheadquarter.com`).

---

## 1. Deploy Baileys Gateway via PM2 (Production Setup)

Since Baileys communicates directly via the WhatsApp WebSocket protocol, it does **not** require Docker or a headless browser (Chromium), making it extremely lightweight and stable. On production, we run the gateway as a background service managed by **PM2**.

### Step 1: Install PM2 Globally
Connect to your VPS/Server via SSH and install PM2 globally:
```bash
npm install -g pm2
```

### Step 2: Set Up Environment Variables
Navigate to the `whatsapp-gateway` directory on your server and ensure the `.env` file matches the API keys of the CRM:
```ini
WHATSAPP_GATEWAY_PORT=8090
WHATSAPP_GATEWAY_HOST=127.0.0.1
WHATSAPP_GATEWAY_API_KEY=bimaheadquarter-openwa-3mP4sV8qL2nR5aT1w
```
*(Alternatively, the gateway will automatically read the shared `.env` from the CRM parent folder if no gateway-specific `.env` is present).*

### Step 3: Install Dependencies
Run the installation inside the gateway folder:
```bash
cd whatsapp-gateway
npm install
```

### Step 4: Start the Gateway with PM2
Launch the gateway using PM2:
```bash
pm2 start server.js --name "whatsapp-gateway"
```

### Step 5: Configure Automatic Startup
To ensure the gateway starts up automatically if the server reboots:
```bash
pm2 startup
pm2 save
```

Verify it is running:
```bash
pm2 status
pm2 logs whatsapp-gateway
```

---

## 2. Configure InsureDesk CRM Environment Variables

Update your CRM's production `.env` file on `bimaheadquarter.com`:

```ini
# WhatsApp Baileys Gateway Configuration
WHATSAPP_GATEWAY_URL=http://localhost:8090
WHATSAPP_GATEWAY_API_KEY=bimaheadquarter-openwa-3mP4sV8qL2nR5aT1w

# Cron Worker Secret (for security)
CRON_SECRET=your_secure_cron_secret_here
```

After updating the `.env` file, rebuild and restart the Next.js server on production:
```bash
npm run build
npm run start
```

## 3. Link Your WhatsApp Device (Scan QR)

1. Open your CRM on [https://bimaheadquarter.com](https://bimaheadquarter.com) and log in.
2. Navigate to the **Operations Hub** and click on **WhatsApp Setup** (or go to `/operations/whatsapp-setup`).
3. The dashboard will show the **QR Code** generated by the Baileys gateway.
4. Open WhatsApp on your phone, go to **Linked Devices** -> **Link a Device**, and scan the QR code.
5. Once scanned, the status will automatically update to **🟢 Connected** (typically within 5 seconds) and persist across server reboots inside `whatsapp-gateway/sessions/`.

---

## 4. Test Message Sending

1. Under the **Send Test Message** panel on the settings page, enter a recipient phone number (including country code, e.g., `919876543210` for India, without '+' or spaces).
2. Enter a short message and click **Send Test Message**.
3. Verify if the recipient receives the message.

## 5. Set Up Cron Automations

To automatically send birthday greetings and renewal notices, configure two cron schedules (using a cron provider, Vercel cron, or a local system crontab on the VPS):

### 1. Queue Generator (Runs Daily at 9:00 AM IST / 3:30 AM UTC)
Triggers birthday scans and renewal scans, creating `PENDING` items in the database queue:
```bash
curl -X GET "https://bimaheadquarter.com/api/cron/whatsapp-worker?secret=your_secure_cron_secret_here&runScans=true"
```

### 2. Queue Processor (Runs Every 5-10 Minutes)
Sequentially processes pending messages in the queue, implementing strict anti-spam delays (8-12 seconds spacing between sends):
```bash
curl -X GET "https://bimaheadquarter.com/api/cron/whatsapp-worker?secret=your_secure_cron_secret_here"
```
